Blogbot

Install Blogbot

Overview

Blogbot is a Google Docs add-on that handles two tasks for blog writers: QA and uploads. It runs inside the Google Doc you're working in as a sidebar with two tabs:

  • QA tab: Runs automated checks across the draft, covering style, links, alt text, plagiarism, AEO structure, legal flags, and more. Checks can be toggled on or off individually.
  • Upload tab: Prepares the doc for CMS upload.

Use the QA tab once your draft is fully written. Once QA is clear, switch to the Upload tab to push the post. Both tabs share an Asana task URL field — pasting the task URL pulls the primary keyword, secondary keywords, brief notes, and live post URL into Blogbot, which several QA checks rely on.

Getting Started

Installing Blogbot

Install Blogbot from the Google Workspace Marketplace. Once installed, open it from any Google Doc by clicking the Blogbot icon in the right-hand sidebar.

The first time you open Blogbot in a doc, Google will ask you to grant permissions. Blogbot needs access to read and modify the active doc, fetch URLs (for link checking and Asana), and store your settings.

Connecting an Asana Task

At the top of the sidebar, paste the full URL of the Asana task for the post you're working on. Once pasted, Blogbot pulls the primary keyword, secondary keywords, task description, and live post URL from the task. You can also enter this information manually.

Several QA checks depend on this data. If the Asana URL isn't connected, those checks run with reduced accuracy.

The QA Tab

The QA tab checks a draft against HubSpot's editorial and AEO/SEO standards.

Running a Scan

  1. Make sure the QA tab is selected at the top of the sidebar.
  2. Paste the Asana task URL (or enter task information manually).
  3. Toggle your scan checks by clicking the down arrow on the Run scan button. For most scans, all checks are recommended.
  4. Click Run scan.

Most scans finish in 30–90 seconds depending on draft length, link count, and which checks are enabled. The original content (plagiarism) and functional links checks are the slowest.

Viewing Results

If the scan finds issues, it returns one or more issue cards sorted by check type. Click a card to jump to the location in the doc where the issue occurs. Some cards offer an auto-fix option or a fix suggestion.

To close a card, click the X in the top right corner — note this does not resolve the issue in the document. After a scan, you can also click Copy results to copy a plaintext summary of all flagged issues to your clipboard.

Blogbot doesn't auto-refresh results when you edit the document. After fixing issues, click Run scan again. You can also re-scan a single check by clicking the rescan button (↺).

Dismissing vs. Fixing

Not every flag needs a fix. Common cases for dismissing rather than editing:

  • A blocked phrase (Authentic Language) is genuinely the right word in context.
  • An AEO structure gap is intentional (e.g., a personal essay format).
  • A legal flag is on quoted source material that's been cleared.

Use your judgment. If you're unsure whether to dismiss a flag, ask your manager or the BCAP Slack channel.

The Upload Tab

The upload tab prepares the draft for upload to the CMS.

How to Prep and Upload

  1. Switch to the Upload tab at the top of the sidebar.
  2. Confirm your upload options:
    • Table of contents: Generates a TOC from the doc's H2s and inserts it near the top of the post. Turn off for short posts, listicles where H2s are the list items, or any post where a TOC would feel redundant.
    • Editor's note: Adds an editor's note inserted at the bottom of the published post (used for updated/refreshed content).
  3. Click Upload doc. Blogbot converts the document to HTML and compresses the document's images.
  4. Your browser will download the compressed images automatically. Save these to your desktop, then upload them to the file manager in 53.
  5. Copy the source code from Blogbot and paste it into the 53 source code window.
Important: Upload your images before pasting the source code into 53. Otherwise, images may not load properly in the blog post body.
Backup: If the Upload tab isn't working, use Blogbot 4 as a fallback.

Settings

Click the settings button at the bottom of the sidebar to open the Blogbot settings menu.

  • Cache toggle: By default, Blogbot caches results from previous scans. If a follow-up scan runs on the exact same text, it fetches results from the last scan. Keep this on except for debugging. Cache is per-doc and resets when you open a different doc.
  • Scan scope: Choose between Current tab only (scans only the open document tab) or Full doc (scans all tabs in the document).
  • Style sub-check toggles: The Style check includes several sub-checks that can each be toggled on and off. Unless debugging, all style sub-checks should be enabled.

Reporting Issues

If Blogbot gives you an error or takes too long to run a scan or upload, let Jamie know. Screenshots are helpful.

If you see a persistent error, try removing and reinstalling Blogbot a couple of times. If it's still not working, check the following:

  • Are you logged in to your HubSpot account?
  • When you reinstalled, did you get an authorization/permissions prompt and click through it?
  • Does the error appear on any doc, or just a specific one? This error will show if you try to open Blogbot on a doc you don't have edit access to.

If you're still seeing the error, reach out to Jamie Juviler (jjuviler@hubspot.com).
