Product Research Assistant

This skill is shared with all HubSpot blog writers — you should have access automatically. To activate it, go to Customize → Skills in Claude and look for it under Shared Skills. Select the skill, then toggle it on in the top right. If you don't see the skill, reach out to your manager.

Current version: v1

Overview

The Product Research Assistant automates competitor product research for listicle-style blog posts. It extracts pricing and features, outputs structured comparison tables, and attaches color-coded verification flags so you know exactly what to manually confirm before publishing.

What It Does

  • Product Research: Gathers key information about competitor tools and products
  • Structured Output: Returns data in clean comparison tables ready for your article
  • Verification Flags: Color-codes each data point by confidence level (see below)
  • Source Citations: Lists the sources used for each data point

Verification Flag System

  • 🟢 Green: Data from official product page, clear and recent — safe to use
  • 🟡 Yellow: From secondary pages or needs verification — check before publishing
  • 🔴 Red: Not found, requires contacting sales, or conflicting data — do not publish without manual verification

When to Use

  • Listicle posts comparing 5–11 products ("Best CRM Tools for Small Business")
  • "Best X" articles where you need consistent data across multiple tools
  • Tool roundups requiring pricing, feature sets, and differentiators
  • Any article where manually researching 8+ products would take hours

Installation

  1. Download the skill: Click the button above to access the skill package
  2. Open Claude Settings: Go to claude.ai → Settings → Capabilities → Skills
  3. Install the skill: Drag and drop the zip file into the Skills upload area
  4. Activate the skill: Enable "Product Research Assistant" in your skills list

How to Use

  1. Tell Claude the article topic and list of products to research
  2. Specify what data you need (pricing tiers, key features, integrations, etc.)
  3. Claude uses the Product Research Assistant skill to gather and structure the data
  4. Review the output, check all 🟡 and 🔴 flags manually before publishing

Tips & Best Practices

Do This

  • Work conversationally with Claude to refine the product list — add, remove, or swap products as needed
  • Always manually verify 🟡 and 🔴 flagged data before publishing
  • Check pricing pages directly — pricing changes frequently
  • Use official product websites as your source of truth
  • Save the source URLs the skill returns for your own reference

Avoid This

  • Publishing any 🔴 flagged data without manual verification
  • Using data that the skill flags as outdated
  • Skipping the manual pricing verification step — always confirm on live pages
  • Relying on third-party review sites for pricing data (G2, Capterra)
